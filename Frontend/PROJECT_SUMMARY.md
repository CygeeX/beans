## 🎨 页面设计说明

### 设计原则
1. **无技术术语**：所有文案使用通俗易懂的语言
2. **清晰的视觉层次**：重要信息突出显示
3. **友好的交互反馈**：每个操作都有明确的反馈
4. **容错性设计**：错误可恢复，不需要单独错误页面

### 视觉风格
- **配色方案**：紫色渐变背景 + 白色卡片
- **字体**：系统默认字体（跨平台兼容）
- **图标**：使用 emoji（无需额外资源）
- **动画**：平滑过渡效果（0.3s）

### 响应式设计
- 桌面端：最大宽度 800px（上传页）/ 1200px（结果页）
- 移动端：自适应布局，触摸友好

## 🔄 用户操作流程

### 流程 1：快速预测
```
访问系统 → 选择"快速预测" → 上传 ground.zip →
点击"开始分析" → 等待处理 → 自动跳转到结果页 →
查看结果 → 下载报告
```

### 流程 2：训练模型
```
访问系统 → 选择"训练模型" → 上传 ground.zip + yield.xlsx →
点击"开始分析" → 等待训练 → 自动跳转到结果页 →
查看结果 → 下载报告
```

### 流程 3：查看历史
```
点击"地块管理分析" → 查看历史记录列表 →
点击"查看详情" → 查看特定结果 → 下载报告
```

## 🔌 API 接口映射

### 后端 API → 前端功能

| 后端接口 | 前端功能 | 触发位置 |
|---------|---------|---------|
| `POST /train` | 训练模型 | 上传页面 - 训练模式 |
| `POST /predict` | 预测产量 | 上传页面 - 预测模式 |
| `GET /result/{run_id}/{filename}` | 下载文件 | 结果页面 - 下载按钮 |
| `GET /health` | 健康检查 | （可选）页面加载时 |

### 请求参数映射

| 后端参数 | 前端字段 | 说明 |
|---------|---------|------|
| `ground` | `files.ground` | 地面观测数据包 |
| `yield_xlsx` | `files.yieldXlsx` | 真实产量记录表 |
| `layout` | `files.layout` | 布局文件（可选）|

## ⚠️ 注意事项

### 1. 浏览器兼容性
- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

### 2. 文件大小限制
- 建议单个文件不超过 100MB
- 具体限制由后端配置决定

### 3. 网络要求
- 需要稳定的网络连接
- 大文件上传可能需要较长时间

### 4. 数据安全
- 历史记录存储在浏览器本地
- 清除浏览器数据会丢失历史记录
- 建议定期下载重要结果

## 🐛 已知问题和限制

### 当前限制
1. **历史记录**：仅存储在本地浏览器，不支持跨设备同步
2. **并发分析**：一次只能进行一个分析任务
3. **文件预览**：不支持在线预览文件内容
4. **批量操作**：不支持批量上传和分析

### 未来改进方向
1. 添加用户登录和云端存储
2. 支持数据可视化图表
3. 添加批量分析功能
4. 支持导出 PDF 报告
5. 添加移动端 App
