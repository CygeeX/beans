## 估产模型到决策建议生成流程

```
输入数据（用户上传）
    │  ground.zip（NDVI/LAI/EVI等遥感特征）
    │  yield.xlsx（历史实测产量）
    │  input/layout.csv（田块布局）
    ▼
特征构建（35维去冗余特征）
    │  src/models.py: transform_feature_table()
    │  SelectKBest(f_regression, k=12) 特征筛选
    ▼
估产模型预测（Exp3 SVR_RBF）
    │  TransformedTargetRegressor + StandardScaler
    │  LOOCV 留一法评估（n=15，评估口径）
    ▼
输出产量与等级
    │  连续预测产量（kg/100株）
    │  高/中/低产等级（top-k=5 / bottom-k=5 / middle）
    ▼
规则匹配（RuleEngine）
    │  查询 kb.sqlite.rules（is_active=1）
    │  条件字段：grade / spad_last / height_mean_slope /
    │           growth_stage / field_type / drought_risk 等
    │  从 kb.sqlite.thresholds 取定量阈值
    ▼
生成建议文本
    │  advice_type：处方（强制性操作建议）/ 提示（参考性建议）
    │  每条建议附 evidence_id + source_id（可追溯）
    ▼
管理建议_中文.json → 前端/API 展示
```

## 一、文档概述

本文档说明豆丰智测大豆产量智能评估与田间管理决策系统中，决策建议知识库的完整构建机制、估产模型与建议生成的关联方式，以及知识库数据库的设计结构。

系统决策建议的生成来自两个并行模块：

**模块一：知识驱动的规则库**
从具有权威性的公开农业技术文件中，下载原文、解析正文、抽取结构化证据条目，生成规则与阈值，最终以 SQLite 数据库和 CSV 主表形式落地存储。

**模块二：机器学习模型输出**
以大豆田块遥感特征为输入，通过支持向量回归模型（SVR_RBF，Exp3，35维去冗余特征）预测批次田块的相对产量，并将预测结果的等级（高/中/低产）和传感器导出字段（SPAD、株高增长率等）作为触发信号，与规则库联动输出个性化田间管理建议。

两个模块共同驱动 `管理建议_中文.json` 的生成，所有建议均具备完整的证据来源链路。

## 三、来源文件获取方式

知识库来源文件通过 `scripts/build_knowledge_base.py` 脚本自动下载并本地化存储。主要流程如下：

1. **URL 清单定义**：脚本内维护 `SOURCES` 列表，每条记录包含 `source_id`、`source_url`、`file_name`、`subdir` 等字段，共 12 条

2. **HTTP 下载**：以 Requests 库发送 GET 请求，模拟浏览器请求头（User-Agent: Chrome/124）防止 403 拒绝，超时 30 秒；SSL 验证失败时尝试跳过验证

3. **本地保存**：文件保存路径为 `knowledge_base/raw_sources/{subdir}/{file_name}`，其中 `subdir` 按来源机构分类（`hlj`/`moa`/`extension`/`fao`）

4. **下载状态记录**：写入 `knowledge_base/source_master.csv` 和 `download_manifest.csv`，记录下载状态、文件大小、耗时等字段；下载失败时写入 `.failed.txt` 占位文件

5. **SQLite 同步**：同步写入 `kb.sqlite` 的 `sources` 表，字段包含下载时间戳

**实际下载结果**：12/12 个文件全部下载成功，8 个 HTML 文件和 4 个 PDF 文件均已完整保存。

## 四、HTML 与 PDF 解析方式

### 4.1 HTML 正文解析

HTML 解析由 `scripts/parse_and_extract_kb.py` 的 Step 1 完成，核心流程：

1. **读取字节流**：使用 `pathlib.Path.read_bytes()` 读取原始字节，由 BeautifulSoup 自动检测编码
2. **标签清洗**：移除对正文无用的标签：`script`、`style`、`nav`、`header`、`footer`、`noscript`、`iframe`、`svg`、`button`、`form`
3. **文本提取**：调用 `soup.get_text(separator="\n", strip=True)` 提取纯文本
4. **空白字符标准化**：
   - 全角空格（`\u2003`、`\xa0`、`\u3000`）→ 普通空格
   - 连续空格/制表符 → 单空格
   - 超过 2 个连续空行 → 2 个空行
5. **保存**：保存至 `knowledge_base/parsed_text/<source_id>.txt`，UTF-8 编码

**HTML 解析结果**：8 个 HTML 来源全部成功解析，字符数从 3,051（MOA-002 索引页）到 11,411（EXT-001）不等，总字符量约 49,000 字符。

### 4.2 PDF 正文解析

PDF 解析通过 `scripts/update_kb_from_pdfs.py` 使用 PyMuPDF（fitz）库完成：

**文本层检测**：读取 PDF 前 5 页，若总字符数不足 100，判定为图像扫描件（无文本层）。

**三种处理方式**：

| 文件 | 页数 | 策略 | 结果 |
|------|:---:|------|------|
| HLJ-004（2026病虫草鼠害PDF） | 316页 | 检测到无文本层（图像扫描件）→ 放弃提取 | 不可解析 |
| HLJ-007（2025科学施肥方案） | 12页 | 全文提取 | 成功，6,147字符 |
| HLJ-005（2025农业主推技术） | 515页 | 按"大豆"关键词筛选相关页面（129页）→ 提取 | 成功，76,550字符 |

## 六、规则与阈值生成机制

### 6.1 规则设计原则

每条规则（rule）对应一条或多条证据的农学操作建议，具备以下属性：

- **rule_id**：命名格式 `R-{类别}-{编号}`，类别含义：F=施肥，N=根瘤菌，W=水分，P=病虫害，M=中后期管理，G=通用
- **trigger_type**：触发类型，分四类（见下节）
- **condition_expr**：触发条件声明式表达式（伪代码），由应用层解析执行
- **advice_type**：处方（直接操作建议，有定量数据支撑）/ 提示（参考性建议，需用户结合实际）
- **is_active**：激活状态，可实现规则热启停，无需修改代码

### 6.2 规则触发类型

| 触发类型 | 数量 | 依赖字段 | 说明 |
|---------|:---:|---------|------|
| 生育期提醒 | 10 | `growth_stage` | 仅依赖生育期，不需要模型输出，可立即实现 |
| 纯规则触发 | 6 | `field_type`、`precipitation_excess`、`rotation_type`、`drought_risk` 等 | 依赖地块属性和气候字段 |
| 模型等级+规则联合 | 3 | `grade` + `spad_last`/`height_mean_slope` | 同时依赖模型预测结果和传感器数据 |
| 模型等级触发 | 2 | `grade` | 仅依赖模型预测的高/中/低产等级 |

## 七、估产模型与决策建议的关联机制

### 7.1 估产模型简介

**模型方案**：Exp3 SVR_RBF（支持向量回归，径向基核函数）

| 参数 | 值 |
|------|:---:|
| 特征维度 | 35（经去冗余处理，来自遥感提取的原始36维特征去除1个冗余特征）|
| 特征筛选 | `SelectKBest(f_regression, k=12)` |
| 目标变量变换 | `TransformedTargetRegressor + StandardScaler` |
| 超参数 | C=10, γ=0.2, ε=0.01（LOOCV 网格搜索最优） |
| 评估方式 | LOOCV（留一法，n=15） |

**LOOCV 展示口径指标（基于 n=15 评估，仅供参考）：**

| 指标 | 值 |
|------|:---:|
| 产量等级识别准确率 | 40.0%（对比随机基线 33.3%）|
| 产量等级 Macro F1 | 0.400 |
| 高产田命中率（top-5）| 40.0%（2/5）|
| 低产田命中率（bottom-5）| 40.0%（2/5）|
| 中位绝对误差 | 0.187 kg/100株 |
| 归一化 RMSE | 20.7% |

**说明**：上述指标基于 n=15 的极小样本 LOOCV 评估，95% 置信区间极宽（准确率约 [0.16, 0.68]），结果具有较高不确定性，仅反映当前样本下的相对排序能力，不能外推为绝对精度。系统已在用户展示界面注明此限制。

### 7.2 展示口径选择

经过与纯分类模型（LinearSVC、RBF-SVC、RidgeClassifier）的 LOOCV 对比实验，系统继续采用"回归→等级"方案，理由：

1. LinearSVC 准确率（53.3%）虽高于回归基线（40%），但高产田命中率大幅退化（20% vs 40%），且差异在 n=15 下统计上不显著（95% CI 大幅重叠）
2. 回归模型可同时输出连续产量数值，RBF-SVC 分类器在相同参数下完全失效（13.3%，低于随机基线 33.3%）
3. 回归模型输出连续产量，支持 `median_absolute_error` 和 `normalized_rmse_mean` 两项展示指标的计算

### 7.3 模型输出如何驱动规则

模型输出以下字段，作为规则引擎的输入信号：

| 字段 | 来源 | 用于触发的规则类型 |
|------|------|----------------|
| `grade`（高产/中产/低产）| 预测产量批次内排名 | 模型等级触发、模型等级+规则联合 |
| `spad_last`（叶片 SPAD 均值）| 传感器特征 | 模型等级+规则联合（阈值 TH-004: ≤P30） |
| `height_mean_slope`（株高增长率）| 传感器特征 | 模型等级+规则联合（阈值 TH-008: >P70） |
| `growth_stage`（当前生育期）| 用户输入/系统推算 | 生育期提醒 |

**等级划分规则**（依据 n=15 批次）：
```
k = min(5, max(1, n // 3))   # n=15 → k=5
预测产量降序 top-k 块  → 高产
预测产量降序 bottom-k 块 → 低产
其余 n-2k 块            → 中产
```

## 八、决策建议生成流程

### 8.1 当前已实现流程（v1，代码驱动）

现有系统中，建议生成逻辑位于 `src/io_utils.py` 的 `make_advice_rules()` 函数，在 `pipeline.py` 的 `run_train()` 和 `run_predict()` 中调用：

1. 合并训练/预测 DataFrame（`df_train_like`）与预测结果（`df_pred_cn`）
2. 计算批次内 SPAD 第 30 百分位数（`spad_low = np.nanpercentile(spad_last, 30)`）
3. 计算批次内株高增长率第 30 百分位数（`htrend_low`）
4. 对每块田按 4 种分支输出建议：
   - **建议A**：低产 AND spad_last ≤ spad_low → 叶色偏淡追肥提示
   - **建议B**：低产 AND height_trend ≤ htrend_low → 株高偏低田间管理提示
   - **建议C**：低产（其他）→ 通用低产提示
   - **建议D**：非低产 → 长势正常提示
5. 最多输出 2 条建议（`tips[:2]`），写入 `管理建议_中文.json`

**当前 v1 系统局限**：覆盖 4 种建议类型，无来源引用，阈值为批次内分位数（无外部农学依据），生育期覆盖仅限于模型输出阶段（收获前评估），高产与中产田输出相同的"长势正常"建议。

### 8.2 目标接入流程（v2，知识库驱动）

规则库 v1.0 已完整构建，设计供未来版本接入：

1. 系统以 `growth_stage`、`grade`、地块属性（`field_type`、`rotation_type`、`drought_risk` 等）为查询键
2. 从 `kb.sqlite.rules` 中检索 `is_active=1` 且匹配 `condition_expr` 的规则
3. 从 `kb.sqlite.thresholds` 获取对应定量阈值
4. 直接输出 `rules.advice_text` 作为建议文本，附带 `evidence_id` 和 `source_id` 供追溯
5. 按 `advice_type`（处方/提示）优先级排序，写入 `管理建议_中文.json`
6. 可选：记录至 `advice_log` 表，支持建议效果追踪

## 九、决策建议知识库数据库设计

### 9.1 技术选型

**存储方式：SQLite 单文件数据库 + CSV 辅助主表（双轨同步存储）**

| 存储形式 | 文件 | 用途 |
|---------|------|------|
| SQLite 数据库 | `knowledge_base/kb.sqlite` | 结构化查询、外键约束、表间关联 |
| CSV 主表 | `knowledge_base/source_master.csv` | 来源元数据，供 pandas 直接读取 |
| CSV 主表 | `knowledge_base/evidence_master.csv` | 证据条目，供批量处理 |
| CSV 主表 | `knowledge_base/rule_master.csv` | 规则定义，供规则引擎读取 |
| CSV 主表 | `knowledge_base/rule_threshold.csv` | 阈值参数，供规则执行 |

**选型说明**：系统为单机部署，无并发写入需求，数据规模为百行级。SQLite 完全满足需求，无需 MySQL、PostgreSQL 或 MongoDB 等独立数据库服务。CSV 格式便于人工审查和编辑，并与 SQLite 同步写入，互为备份。

## 十一、决策建议的可追溯性设计

### 11.1 四级可追溯链

每条输出的田间管理建议，均可通过以下四级链路回溯至原始来源文件：

```
建议文本（管理建议_中文.json 中的 advice_text）
    │ rule_id
    ▼
规则定义（kb.sqlite.rules）
    │ evidence_id（分号分隔）
    ▼
证据条目（kb.sqlite.evidence）
    │  evidence_text：原文引用
    │  normalized_action：标准化操作
    │ source_id
    ▼
来源文件（kb.sqlite.sources）
    │  source_title：文件标题
    │  publisher：发布机构
    │  source_url：原始网页地址
    │  local_path：本地存储路径
    ▼
原始文件（knowledge_base/raw_sources/<subdir>/<file_name>）
```

### 11.2 证据等级与建议强度

| 证据等级 | 建议类型 | 说明 |
|---------|:-------:|------|
| A + 定量数据 | **处方** | 有具体剂量/品种/时间等定量信息，来源为省级官方或国家级部门 |
| A + 原则性描述 | 提示 | 操作方向明确，无精确定量，来源权威 |
| B（国际参考）| 提示 | 可作为辅助参考，需结合本地实际情况调整 |
| 批次内分位数触发 | 提示 | TH-004（P30）、TH-008（P70）无外部农学依据，为启发式设计，置信度说明明确标注 |
